# 闲鱼卖家工具链 (Xianyu Seller Toolkit)

> 所有工具只做**内容层**，不碰平台操作。

## ⚡ 核心原则：按需调用

**被触发时，必须先判断用户意图，只执行需要的步骤，不要总是跑全流程。**

## 🧠 智能路由表

根据用户的话，匹配对应工具：

| 用户说了什么 | 调用哪个工具 |
|-------------|-------------|
| "处理这个资料""准备上架""完整走一遍" | → 完整管线（流程A） |
| "润色一下""排版优化""格式化" | → 只用 `baoyu-format-markdown` |
| "生成展示图""做几张卡片""商品图" | → 只用 `baoyu-image-cards` + `baoyu-imagine` |
| "生成封面图""做张图" | → 只用 `baoyu-imagine` 或 `FishClaw generate_image_tools` |
| "转成HTML""HTML交付版""浏览器打开" | → 只用 `baoyu-markdown-to-html` + 使用说明 |
| "写文案""宝贝描述""闲鱼标题" | → 只用 `FishClaw prompt_tools` |
| "竞品调研""看看闲鱼上怎么卖的""定价参考" | → 只用 `ai-goofish` |
| "社交卡片""小红书图文" | → 只用 `guizang-social-card-skill` |
| "生成使用说明""操作手册" | → 只用 `生成使用说明` |

## 工具清单

| # | 工具 | 做什么 | 输入 | 输出 |
|---|------|--------|------|------|
| 1 | `baoyu-format-markdown` | 润色排版 + 优化标题 | .md 文件 | 润色版 .md |
| 2 | `baoyu-image-cards` + `baoyu-imagine` | 内容→5张商品展示图 | .md 文件内容 | 5张 PNG (1080×1440) |
| 3 | `guizang-social-card-skill` | Markdown→社交卡片 | .md 文件 | PNG 卡片 |
| 4 | `baoyu-markdown-to-html` | .md→精美HTML | .md 文件 | .html (浏览器双击打开) |
| 5 | `FishClaw prompt_tools` | AI写闲鱼文案 | 商品主题 | 标题+描述+卖点+话术 |
| 6 | `FishClaw generate_image_tools` | AI生成封面图 | 主题描述 | PNG 图片 |
| 7 | `ai-goofish` | 闲鱼关键词监控+竞品分析 | 关键词 | 竞品数据 |
| 8 | `生成使用说明` (内置) | 生成顾客使用手册 | 交付版内容分析 | `使用说明.txt` |

## 流程 A：完整管线

仅当用户明确要求**完整处理**或**准备上架**时执行：

```
                        原始 .md 文件
                              │
 ┌────────────────────────────┼────────────────────────────┐
 │           📝 内容层                          🛒 销售层   │
 │                                                            │
 │  [1] baoyu-format-markdown                                │
 │      分析内容 → 选标题 → 润色排版                           │
 │      → 润色版.md                                           │
 │                                                            │
 │  [2] baoyu-image-cards + baoyu-imagine                    │
 │      AI分析内容 → 确认风格 → 写5个提示词 → 逐个出图          │
 │      → 01-cover.png  02-value.png  03-preview.png          │
 │        04-top.png    05-cta.png                             │
 │                                                            │
 │  [3] 去掉商品信息区（🛒闲鱼宝贝信息段落）                     │
 │      → 交付版.md                                           │
 │                                                            │
 │  [4] baoyu-markdown-to-html (modern + blue)               │
 │      → 交付版.html (浏览器双击打开)                         │
 │                                                            │
 │  [5] 生成使用说明（内置）                                    │
 │      分析内容结构 → 写零门槛操作指南                         │
 │      → 使用说明.txt                                        │
 │                                                            │
 ├────────────────────────────────────────────────────────────┤
 │                                                            │
 │  [6] FishClaw prompt_tools                                 │
 │      根据主题生成闲鱼优化文案                                │
 │      → 闲鱼宝贝描述.md                                      │
 │      （标题 / 描述 / 卖点 / 自动回复话术 / 标签）             │
 │                                                            │
 └────────────────────────────────────────────────────────────┘
                              │
                              ▼
 ╔═══════════════════════════════════════════════════════════╗
 ║                    📦 最终交付清单                          ║
 ╠═══════════════════════════════════════════════════════════╣
 ║  🖼️  5张展示图 PNG         → 上传到闲鱼                     ║
 ║  📝 闲鱼宝贝描述.md        → 复制粘贴到闲鱼                  ║
 ║  📄 润色版.md              → 自己留底                        ║
 ║  📦 交付版.html             → 发给顾客 ⭐                    ║
```

## 步骤 [5] 详细规范：生成使用说明

根据交付内容分析后，生成 `使用说明.txt`，遵循以下模板结构：

```text
=== 《[资料名称]》使用说明 ===

📂 怎么打开？
  双击「[文件名].html」，用浏览器（Chrome/Edge/Safari）打开。

🧭 怎么浏览？
  [根据实际内容描述导航方式，如：左侧有分类目录，点一下跳过去]

🔗 链接怎么用？
  [根据实际内容描述链接，如：每个项目名都是蓝色链接，点一下直接打开GitHub]

🔍 怎么搜索？
  浏览器里按 Ctrl+F（Mac 按 Command+F），输入关键词，直接定位。

💡 小技巧
  [根据内容类型给 2-3 条实用建议，如：可以加入浏览器收藏夹、手机也能打开等]

❓ 有问题？
  闲鱼聊天直接找我，看到就回～
```

**核心要求**：语言小白化，零术语，假设顾客只会双击文件。
